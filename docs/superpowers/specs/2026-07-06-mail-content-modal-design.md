# 邮件内容弹层设计

## 目标

在供应商邮件页面的邮件列表中增加“查看”操作。用户点击后，以弹层显示邮件元信息和正文。正文优先显示 HTML，HTML 缺失时回退为纯文本。本次不展示或下载附件。

## 数据设计

扩展 `MailMessageView`，增加可空的 `TextBody` 和 `HtmlBody` 字段。`MailCacheService` 将邮件服务商已经获取的正文投影到该视图中，现有轮询响应随列表数据一并返回正文。

页面当前最多展示 10 封邮件，因此直接随轮询响应返回正文，避免为详情增加独立接口、缓存查询和额外网络往返。

## 页面交互

邮件表格增加“操作”列，每行提供“查看”按钮。点击按钮后打开一个 Bootstrap modal，显示：

- 发件人
- 主题
- 发送时间
- 邮件正文

弹层可通过关闭按钮、遮罩和 Escape 键关闭。筛选、自动轮询、手动刷新和验证码复制行为保持不变。

## 正文渲染与安全

若 `HtmlBody` 有内容，将其写入带 `sandbox` 属性且不授予脚本、表单、弹窗或顶层导航权限的 `iframe.srcdoc`。iframe 隔离邮件正文的样式，避免影响宿主页面；外部资源是否加载由浏览器及现有站点安全策略决定。

若 HTML 正文为空，则将 `TextBody` 作为纯文本写入 DOM，并保留换行。若两种正文都为空，显示本地化的“无邮件内容”提示。

所有元信息均使用文本节点或现有 HTML 转义函数写入，不能作为 HTML 执行。附件元数据不进入弹层。

## 本地化

为中英文资源增加“操作”“查看”“邮件内容”“无邮件内容”等界面文本，保持两种语言资源键一致。

## 测试与验收

- `MailCacheService` 投影测试验证纯文本和 HTML 正文均被保留。
- 页面模型/响应测试验证轮询结果包含正文。
- 现有测试套件全部通过。
- 浏览器验证查看按钮能打开弹层，HTML 优先、纯文本回退正确，关闭方式可用。
- 验证邮件筛选、刷新、自动轮询和数字复制功能没有回归。

## 非目标

- 附件展示或下载。
- 独立邮件详情接口。
- 邮件回复、转发、删除或已读状态管理。
- 对远程图片做代理或下载。
